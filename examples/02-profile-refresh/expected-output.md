# Expected output for Example 02

This story should show how a confirmed profile stays fresh over time.

## What should stand out

- the same person stays the same entity across runs
- the profile summary becomes slightly more specific after the new note
- the alias list grows if needed
- the profile health output should still be readable and manageable

## Why this matters

- it proves the system is not only creating people, but maintaining them
- it shows how new information becomes a durable update instead of a new record
