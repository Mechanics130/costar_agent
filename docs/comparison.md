# CoStar Comparison Notes

## Compared with Dex / Clay

| Axis | CoStar | Dex / Clay |
| --- | --- | --- |
| Main job | Durable people context | Relationship CRM / enrichment |
| Data shape | Notes, transcripts, profiles, briefs, graph, views | Contacts and CRM-style enrichment |
| Workflow | Capture -> confirm -> commit -> reuse | Import -> enrich -> manage |
| Core edge | Skill engine and persistent context loop | CRM-style relationship management |
| Deployment | Self-hostable open-core engine | Hosted SaaS first |

## Compared with Notion / Docs

| Axis | CoStar | Notion / Docs |
| --- | --- | --- |
| Main job | Extract and maintain people context automatically | Store and organize information manually |
| Relationship memory | First-class | User-built |
| Briefing support | Built in | Manual |
| Graph / views | Built in | Not native |

## Compared with ChatGPT direct usage

| Axis | CoStar | ChatGPT direct |
| --- | --- | --- |
| Persistence | Durable profile store | Session-based unless manually saved |
| Confirmation | Review and commit flow | User-managed |
| Reuse | Profiles, views, graph, briefing | Prompt-by-prompt |

## Public positioning

CoStar should be described as:

- an open-core skill engine
- a self-hostable relationship context layer
- a durable people-memory system

If you want to use chief-of-staff language, keep it as a one-line metaphor in the main README, not as the core positioning for the engine repo.
