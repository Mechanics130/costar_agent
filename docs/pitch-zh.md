# CoStar 产品介绍

## 100字版

CoStar 是一个开源的 skill 引擎，用来把零散的笔记、会议纪要和转写内容，整理成可长期使用的人物档案、会前简报、关系图谱和持续视图。它不要求用户手工重建上下文，而是帮你把关系记忆保留下来。它适合开发者和产品人先接入，也能在未来承载一个真正面向用户的 AI 幕僚产品。

## 300字版

CoStar 是一个开源的 skill 引擎，目标是把零散的人物信息、会议内容、聊天记录和历史材料，整理成真正能长期使用的关系上下文。

它的核心不是“存资料”，而是“识别资料里的人、确认重要更新、把更新写回档案，并持续维持这些档案可用”。这样下次开会、约谈、协作时，系统不需要重新从零理解对象，而是直接基于最新确认过的关系上下文工作。

CoStar 现在服务两类人：

- 工程师、产品人、二次开发者：可以直接使用开源 skill 引擎
- 未来的终端用户：可以使用建立在 CoStar 上的消费端产品

这个拆分很重要。开源引擎负责能力和可扩展性；消费端产品负责更低门槛、更像“AI 幕僚”的体验。

如果你想要的是一个可自托管、可接入自定义模型、能持续积累人物上下文的底层引擎，CoStar 就是这个底座。

## 500字版

CoStar 是一个开源的 skill 引擎，目标是把零散、杂乱、跨时间的人物材料，整理成可长期使用的关系上下文。

它处理的不是简单文档搜索，而是“人”的问题：谁是谁、谁和谁有什么关系、谁最近有什么变化、谁值得继续跟进、下一次沟通前应该知道什么。为此，CoStar 设计了一条很清晰的主链路：

1. 采集资料
2. 识别人物
3. 提取更新
4. 用户确认
5. 写回档案
6. 刷新持续视图
7. 基于确认档案生成简报、模拟和图谱

这样做的好处是，关系记忆不会只停留在某一份笔记或某一次会议里，而是变成一个可以持续积累、持续校正、持续复用的上下文系统。

CoStar 采用 open-core 的方式发布：仓库里提供的是 skill 引擎本身，适合开发者、产品经理和想要自托管的人使用；消费端产品则可以作为单独的 UI 层叠加在这个引擎之上。这样既能保留工程可控性，也能让真正不想碰命令行的用户获得更简单的体验。

如果你在找一个“人的上下文底座”，一个“会前自动准备信息的 AI 幕僚”，或者一个“可自托管的人脉智能层”，CoStar 就是这类产品的基础引擎。
